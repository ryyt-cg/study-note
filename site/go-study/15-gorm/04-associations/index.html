Associations

Associations are used to define the relationship between two tables. GORM supports the following associations:

Belongs To

A belongs to association sets up a one-to-one connection with another model, such that each instance of the declaring model "belongs to" one instance of the other model.

Notice here that, on the User object, there is both a CompanyID as well as a Company. By default, the CompanyID is implicitly used to create a foreign key relationship between the User and Company tables, and thus must be included in the User struct in order to fill the Company inner struct.

// `User` belongs to `Company`, `CompanyID` is the foreign key
type User struct {
  gorm.Model
  Name      string
  CompanyID int
  Company   Company
}

type Company struct {
  ID   int
  Name string
}

Override Foreign Key

GORM provides a way to customize the foreign key.

type User struct {
  gorm.Model
  Name         string
  CompanyRefer int
  Company      Company `gorm:"foreignKey:CompanyRefer"`
  // use CompanyRefer as foreign key
}

type Company struct {
  ID   int
  Name string
}

Has One

A has one association sets up a one-to-one connection with another model, but with somewhat different semantics (and consequences). This association indicates that each instance of a model contains or possesses one instance of another model.

// User has one CreditCard, UserID is the foreign key
type User struct {
  gorm.Model
  CreditCard CreditCard
}

type CreditCard struct {
  gorm.Model
  Number string
  UserID uint
}

Has Many

A has many association sets up a one-to-many connection with another model, unlike has one, the owner could have zero or many instances of models.

// User has many CreditCards, UserID is the foreign key
type User struct {
  gorm.Model
  CreditCards []CreditCard
}

type CreditCard struct {
  gorm.Model
  Number string
  UserID uint
}

Many To Many

Many to Many add a join table between two models.

// User has and belongs to many languages, `user_languages` is the join table
type User struct {
    gorm.Model
    Name       string
    Email      string
    CompanyId  uint
    Company    Company
    CreditCard CreditCard
    Languages []Language `gorm:"many2many:user_languages;"`
}

type Company struct {
    gorm.Model
    Name string
}

type CreditCard struct {
    gorm.Model
    Number string
    UserID uint
}

type Language struct {
  gorm.Model
  Name string
}

When using GORM AutoMigrate to create a table for User, GORM will create the join table automatically

Preload (Eager Loading)

Preload is used to load the associated data of a model. It will execute a SQL query to load the associated data of the model.

db.Preload("Company").Find(&users)
// SELECT * FROM `company` WHERE `company`.`id` = 1 AND `company`.`deleted_at` IS NULL
// SELECT * FROM `user` WHERE `user`.`id` = 1 AND `user`.`deleted_at` IS NULL ORDER BY `user`.`id` LIMIT 1

Joins Preloading

Preload loads the association data in a separate query, Join Preload will loads association data using left join, for example:

    db.Joins("Company").Joins("CreditCard").First(&joinsUser, 1)

NOTE: Join Preload works with one-to-one relation, e.g: has one, belongs to.

Golang Workspace

Go 1.18 introduced the concept of workspace to create a projects of several modules that share a common list of dependencies through a new file called go.work. The dependencies in this file can span multiple modules and anything declared in the go.work file will override dependencies declared in the go.mod file.

This concept is similar to the concept of a monorepo, where you have a single repository that contains multiple projects. The workspace is a way to manage multiple modules in a single repository.

Pre-requisites

  • Install Go 1.18 or later

Demo

  1. Create a workspace directory
mkdir -p 01-workspace
cd 01-workspace
mkdir -p calculator
mkdir -p service-app

// Initialize the go module
cd calculator
go mod init calculator

cd ../service-app
go mod init service-appo
  1. Create the workspace
// At 01-workspace directory, initialize the workspace
go work init ./calculator ./service-app
  1. Create a function in the adder module
package calcalator

func Add(a, b int) int {
    return a + b
}
  1. Create a function in the service-app module
package main

import (
    "fmt"
    "calculator"
)

func main() {
    fmt.Println(calculator.Add(1, 2))
}
  1. Add the hello module to the go.work file
// At 01-workspace directory
go work use ./hello
  1. Run the service-app module
// At 01-workspace directory
go run ./service-app
  1. Run the hello module
// At 01-workspace directory
go run ./hello

Getting Started

This is a getting started guide for the Atlas project.

Pre-requisites

  • install atlas
  • install sqlite3

Demo

Create a new database file

sqlite3 mydb.db

Add table customers to the database

CREATE TABLE customers (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    email TEXT NOT NULL
);

Step 1: Inspecting the database

Usage

atlas schema inspect [flags]

Example: This command inspects the database and outputs the schema in HCL format.

atlas schema inspect --url sqlite3://mydb.db > schema.sql

schema.hcl

table "customers" {
  schema = schema.main
  column "id" {
    null = true
    type = integer
  }
  column "first_name" {
    null = false
    type = text
  }
  column "last_name" {
    null = false
    type = text
  }
  primary_key {
    columns = [column.id]
  }
}
schema "main" {
}

Step 2: Planning a change

Once you have the schema in HCL format, you can plan a change. This is done by modifying the HCL file in a way that it defines the desired state of the database schema. The file should include the changes you want to make to your database schema, such as creating tables, columns, or indexes.

Adding a new table called orders to the schema.hcl file.

table "customers" {
  schema = schema.main
  column "id" {
    null = true
    type = integer
  }
  column "name" {
    null = false
    type = text
  }
  column "email" {
    null = false
    type = text
  }
  primary_key {
    columns = [column.id]
  }
}

table "orders" {
    schema = schema.main
    column "id" {
        null = true
        type = integer
    }
    column "customer_id" {
        null = false
        type = integer
    }
    column "order_date" {
        null = false
        type = date
    }
    primary_key {
        columns = [column.id]
    }
    foreign_key "fk1" {
        columns     = [column.customer_id]
        ref_columns = [table.customers.column.id]
    }
}

Step 3: Applying the change

Final step is to apply the change to the database.

Usage

atlas schema apply [flags]

Example: This command applies the changes to the database.

atlas schema apply --url sqlite3://mydb.db --to file://schema.sql

Verify the change

Verify the change by inspecting the database again.

atlas schema inspect --url sqlite3://mydb.db

Conclusion

This guide provides a quick overview of how to use Atlas to manage your database schema. This method is called Declarative Schema Management. There are other strategies for managing database schema: * Versioned schema management * Automated schema management
